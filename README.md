# UDP Protocol — надёжная передача поверх UDP

Клиент-серверный протокол с контролем целостности (SHA256), подтверждениями доставки и автоматическими ретрансмиссиями.

## Техническое задание

Разработать сетевые сервер и клиент, взаимодействующие по UDP.

**Клиент:** в 10 потоков формирует пакеты данных. Каждый пакет содержит:
- номер пакета (без повторений)
- время формирования
- случайные данные длиной в пределах `<номер пакета>-<2*номер пакета>`
- контрольную сумму SHA256

Все пакеты отправляются по единственному UDP соединению. Общее число пакетов — не менее 10 000. Клиент получает подтверждение об успешном/неуспешном получении каждого пакета и выводит данные строго в порядке увеличения номеров пакетов.

**Сервер:** получает пакеты, проверяет контрольную сумму, подтверждает клиенту успех/неудачу, выводит номер пакета, дату и время формирования, дату и время получения, результат проверки целостности.

**Общий процент потерь не должен превышать 3%.**

## Требования

- Go 1.22+
- make
- Linux (для тестов с искусственными потерями)

## Установка и запуск

### Локально

```bash
git clone https://github.com/FreeSoul777/udp-protocol.git
cd udp-protocol

# Установка инструментов
go install golang.org/x/tools/cmd/goimports@v0.24.0
go install github.com/golangci/golangci-lint/cmd/golangci-lint@v1.62

# Сборка
make build
```

### Запуск сервера

```bash
# С настройками по умолчанию (127.0.0.1:9000)
./bin/server

# С кастомными параметрами
./bin/server --addr=0.0.0.0:8080 --workers=16
```

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--addr` | `127.0.0.1:9000` | Адрес прослушивания |
| `--workers` | `0` (CPU cores) | Количество воркеров |

### Запуск клиента

```bash
# С настройками по умолчанию (10000 пакетов, 10 потоков, 60s таймаут)
./bin/client

# Кастомные параметры
./bin/client --server=192.168.1.10:9000 --packets=50000 --workers=20 --timeout=120s
```

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--server` | `127.0.0.1:9000` | Адрес сервера |
| `--packets` | `10000` | Количество пакетов |
| `--workers` | `10` | Количество потоков |
| `--timeout` | `60s` | Таймаут работы |

### Быстрый старт через Makefile

```bash
# Сервер
make run-server

# Клиент
make run-client
```

Параметры по умолчанию можно изменить в `Makefile`:

```makefile
PACKETS_COUNT = 10000
WORKERS_COUNT = 10
```

## Тестирование

```bash
# Юнит-тесты
make test-unit

# Интеграционный тест (100 пакетов, 2 потока)
make test-smoke

# Нагрузочный тест (10000 пакетов, 10 потоков)
make test-load

# Все тесты
make test-all

# Тесты с 2% искусственных потерь (требует sudo)
make test-with-loss
```

## Docker

Для разработки используется скрипт `scripts/docker-dev.sh`, который обеспечивает синхронизацию изменений с локальной директорией, сборку и тестирование внутри контейнера.

```bash
# Запуск в dev-режиме
make docker-dev
```

При первом запуске собирается образ. При последующих — изменения из локальной директории синхронизируются в контейнер, проект пересобирается и запускаются тесты. После завершения остаётся интерактивный shell внутри контейнера.
